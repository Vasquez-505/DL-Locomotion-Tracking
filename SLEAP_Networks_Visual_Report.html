Neural Network Architectures for Drosophila Leg Contact Tracking

A visual architecture report for the three SLEAP-NN models used in the thesis project: U-Net 64, ImageNet-pretrained ConvNeXt-Tiny, and ImageNet-pretrained Swin Transformer Tiny. The focus is structural understanding, not model ranking.

Shared Input Pipeline

All three networks receive the same preprocessed frame tensor.

The architecture comparison keeps the data path fixed: RGB FTIR frames, geometric-only augmentation during training, batch assembly, and division by 255 before the model forward pass.

Step 1 Ensure RGB

Guarantees a 3-channel input tensor even if a source frame is accidentally greyscale.

Step 2 Size cap + padding

Frames are resized/padded by SLEAP-NN so spatial dimensions are compatible with max_stride=32.

Step 3 Geometric augmentation

Training only: rotation in [-15, +15] degrees and scale in [0.9, 1.1]. No brightness/noise augmentation.

Step 4 Batch assembly

Batch size 6 for train and validation loaders; train loader is shuffled.

Step 5 Normalise

Pixel values are divided by 255, giving values in [0, 1]. No ImageNet mean/std normalisation is applied.

FTIR foot contacts appear as tiny bright regions on a dark background. Brightness augmentation could create artificial contact-like signals, so the augmentation is geometric only.
Shared Output Head

Every backbone ends in the same confidence-map predictor.

The decoder output is converted to nine keypoint heatmaps using a 1 x 1 convolution. The output stride is 4, so a 160 x 480 tensor gives a 40 x 120 output grid.

Confidence-map head

Backbone/decoder feature map H/4 x W/4 x C

1 x 1 Conv: C -> 9 channels

Output: H/4 x W/4 x 9

Each channel predicts the confidence for one leg keypoint. During training, each visible keypoint is represented by a Gaussian blob with sigma=2.5 grid units.

Invisible keypoints

If a keypoint is marked visible = 0, its target confidence map is all zeros. The model is therefore trained to suppress activations for non-contacting legs.

At inference, SLEAP-NN finds the global peak in each confidence map, multiplies by output_stride=4, then maps coordinates back to the original video frame using the stored preprocessing scale.

Model 1

U-Net 64: convolutional encoder-decoder.

11.7M parameters Random initialisation filters=64 filters_rate=1.5 output_stride=4

A U-Net block is not a generic black box: here it means two same-padded 3 x 3 convolutions, each followed by ReLU, with no BatchNorm.

Model 2

ConvNeXt-Tiny: modern convolutional backbone with SLEAP-NN decoder.

87.5M parameters ImageNet-1K pretrained depths [3, 3, 9, 3] widths [96, 192, 384, 768]

A ConvNeXt stage contains repeated ConvNeXt blocks. A ConvNeXt block is an inverted bottleneck: depthwise 7 x 7 convolution, LayerNorm, pointwise expansion C -> 4C, GELU, pointwise projection 4C -> C, residual add.

What is a ConvNeXt block?

Input H x W x C

Depthwise Conv 7x7, groups=C

LayerNorm -> Pointwise C -> 4C -> GELU -> Pointwise 4C -> C

Residual add -> Output H x W x C

The block keeps spatial size and channel count unchanged. The inverted bottleneck expands features internally, then projects them back to the original width.

Model 3

Swin Transformer Tiny: hierarchical shifted-window attention.

~87.25M parameters ImageNet-1K pretrained depths [2, 2, 6, 2] window_size=7

A Swin stage contains W-MSA/SW-MSA Transformer blocks. Patch Merging halves the token grid and doubles channel width by concatenating 2 x 2 neighbouring tokens and projecting 4C -> 2C.

What is a Swin block?

LayerNorm -> Window Multi-head Self-Attention -> residual

LayerNorm -> MLP with expand ratio 4 -> residual

Blocks alternate between W-MSA and shifted-window MSA. The shift is half the window size, allowing information to cross window boundaries efficiently.

U-Net takeaway

Most direct dense-prediction model: local convolutional features, explicit downsample/upsample symmetry, and spatial skip connections.

ConvNeXt takeaway

Modern convolutional hierarchy with large-kernel depthwise blocks, inverted bottlenecks, LayerNorm, and ImageNet-pretrained visual features.

SwinT takeaway

Transformer-style local attention with shifted windows and Patch Merging, adapted to SLEAP-NN with the same stride-4 decoder head.
